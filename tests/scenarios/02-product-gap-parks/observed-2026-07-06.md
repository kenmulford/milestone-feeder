> ## Summary
> Add a "Featured" section to the homepage that renders the catalog items currently marked as featured, using the existing `ProductCardGrid` component. The section consumes the featured-items data/query introduced by #A. This issue covers rendering only, not how items become featured (that policy is out of scope here).
>
> ## Acceptance criteria
> - [ ] Happy path: the homepage shows a "Featured" section that renders the currently-featured catalog items via the existing `ProductCardGrid` component (`src/components/ProductCardGrid.tsx`), matching how any other product listing renders cards.
> - [ ] Empty state: when no catalog items are currently featured, the section renders its empty state (no grid, no cards) instead of an empty grid shell.
> - [ ] Error state: when the featured-items fetch fails, the section renders its error state instead of a partial or broken grid.
> - [ ] Loading/edge state: while the featured-items fetch is in flight, the section renders a loading state (not the empty or error state) until the fetch resolves, so a slow response never flashes a false empty or error state.
>
> ## Design (recorded, consistent)
> Reuse the existing product-listing pattern rather than building a new grid: the Featured section renders through `ProductCardGrid`, the same component used for every other product listing in the app, so its card layout, spacing, and interaction behavior are inherited, not reinvented. The section is read-only display (no destructive action, no form), so no confirm affordance is needed. The empty and error states are the two states the project's conventions mandate for every user-facing surface; the loading state is the standard transient state between "fetch started" and "fetch resolved to populated/empty/error," required so the section never renders a false empty or error before data arrives.
> - Convention followed: `.project/conventions.md#Engineering conventions` ("UI: reuse the existing `ProductCardGrid` component for any product listing. Pattern: `src/components/ProductCardGrid.tsx`"; "States: empty and error states are required on every user-facing surface").
